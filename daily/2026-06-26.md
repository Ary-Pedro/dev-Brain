## Sessão 18:00 — Restauração CampaignWizard + fix template WA

**Tópicos:** feature/waba-integration — wizard campanha, templates WhatsApp, modal de criação

**Problema → Solução:**
- **P:** `CampaignWizard.jsx` (2295L branch) completamente diferente do pre-prod (520L) → **S:** restaurado para base pre-prod + adicionado apenas campos WABA (`provider_type`, `waba_template_name/language/params`, throttle_config) · `frontend/src/pages/campaigns/CampaignWizard.jsx`
- **P:** `WhatsAppMessageEditor.jsx` não setava campos WABA ao selecionar template → **S:** `onChange` inclui `waba_template_name/language/params` quando `providerType === 'waba'` · `frontend/src/pages/campaigns/components/WhatsAppMessageEditor.jsx`
- **P:** `CampaignReport.jsx` sempre mostrava "Campanha não encontrada" ao abrir como modal → **S:** `campaignsApi.get(activeId)` em vez de `campaignsApi.get(campaignId)` (prop não passada) · `frontend/src/pages/campaigns/CampaignReport.jsx`
- **P:** Modal "Criar Template WhatsApp" local mostrava `WabaTemplateEditorModal` (WABA-only, sem toggle) em vez de `WaTemplateFormModal` (toggle Green-API/WABA) → **S:** raiz real: `CampaignsPage.jsx` é o componente da rota `/campaigns/template`, não `TemplatesGalleryPage`; substituído `WabaTemplateEditorModal` por `WaTemplateFormModal` lá · `frontend/src/pages/campaigns/CampaignsPage.jsx`
- **P:** `TemplatesGalleryPage.jsx` (branch 644L) com imports quebrados, `handleModalClose`/`editingTemplateId` indefinidos, canal WhatsApp indo para modal errado → **S:** reescrito do zero: base 100% pre-prod + WABA (wabaTemplatesApi, dedupeTemplates, WaTemplateFormModal, SmsTemplateFormModal, RcsTemplateFormModal) · `frontend/src/pages/campaigns/TemplatesGalleryPage.jsx`
- **P:** Múltiplas iterações erradas corrigindo `TemplatesGalleryPage` enquanto bug estava em `CampaignsPage` → **S:** trace URL → rota → componente antes de editar qualquer arquivo

**Decisões:**
- WABA só entra nos fluxos WhatsApp; SMS/RCS/email mantêm comportamento pre-prod
- `WaTemplateFormModal` é a única entrada p/ criação/edição de template WhatsApp (tem toggle interno GreenAPI/WABA)
- `WabaTemplateEditorModal` removido de `CampaignsPage` e `TemplatesGalleryPage` — estava causando split incorreto

**Pendências:**
- Arquivos não commitados: 22 arquivos (backend WABA + i18n + frontend) — commit pendente sob demanda do user
- Testes unitários não atualizados para novos fluxos
